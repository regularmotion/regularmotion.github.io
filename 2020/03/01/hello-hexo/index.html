Hello, Hexo

Mar 01 2020

Hexo는 Node.js로 개발된 블로그 프레임워크입니다.
마크다운으로 작성한 글을 배포 할 수 있도록 되어 있습니다.
따라서 마크다운과 깃에 익숙한 분은 쉽게 블로그를 구축/운용 할 수 있는 프레임워크입니다.

  • Markdown Guide: https://www.markdownguide.org/basic-syntax/
  • Git Tutorial: https://git-scm.com/docs/gittutorial

1. 설치 - https://hexo.io/ko/docs/

v8.10 이상의 Node.js와 Git이 미리 설치되어 있는 상태에서 hexo-cli 패키지를 설치합니다.
$ npm install -g hexo-cli
Hexo가 설치됐다면, $ hexo -v 명령어를 실행하여 정상적으로 설치 됐는지 확인합니다.

2. 초기화 - https://hexo.io/ko/docs/setup

Hexo가 설치됐다면, 프로젝트 생성 및 필요한 패키지 설치를 위해 아래의 명령어를 실행합니다.

정상적으로 프로젝트 생성 및 초기화가 완료되면 다음과 같은 폴더 구조가 생성됩니다.

  • scaffolds: Scaffold 파일이 위치하는 폴더입니다. Hexo 명령어 (e.g. hexo new post {post-name})로 새로운 포스트나 페이지를 생성 할 때, Scaffolds를 기준으로 새 파일을 생성합니다.
  • source: 웹 사이트의 컨텐츠(Markdown Files)가 위치하는 폴더입니다. 렌더링 가능한 파일(e.g. Markdown, HTML 등)은 전처리 후 public 폴더로 복사됩니다.
  • themes: 설치된 Theme가 저장되는 폴더입니다. 설치된 Theme 중 적용하려는 Theme를 _config.yml 파일에서 지정합니다.

3. 설정 - https://hexo.io/ko/docs/configuration

_config.yml 파일의 속성을 수정하여 블로그/사이트의 설정을 변경 할 수 있습니다.
사이트의 이름, URL, 게시글의 Permalink 형식, Theme, Deploy 속성 등을 설정 할 수 있습니다.

4. 새 글 작성 - https://hexo.io/ko/docs/writing

hexo new post {post-name} 명령어를 실행하면 /source/_post/{post-name}.md 경로에 파일이 생성됩니다.
생성된 파일에 마크다운 양식으로 작성하면 됩니다. 작성된 글은 $ hexo generate --deploy 명령어로 배포 할 수 있습니다.
Post의 Front-matter는 기본적으로 title, date, tags를 포함합니다. 글에 추가하고 싶은 Tag가 있는 경우, Front-matter의 tags에 선언하면 됩니다.

5. Disqus 연동

Hexo로 구축한 사이트에 Disqus를 연동하기 위해서는, 설정한 Theme에서 Disqus 연동을 지원하는지 확인해야 한다.
내가 선택한 Polar Bear Theme의 경우, /themes/{theme-name}/_config.yml 설정 파일에 disqus_shortname 입력만으로 연동이 됐다.
disqus_shortname은 Disqus 가입 후 Site 생성하면 발급된다. https://disqus.com/
